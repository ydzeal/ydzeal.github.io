动态规划及其常见类型

发表于 2017-03-10 | 分类于 Academy

什么是动态规划？

简单来说，动态规划（Dynamic Programming）就是通过递推来解决问题。这是一种将复杂问题转化为相似子问题，并从最小子问题推演至整个问题从而解决问题的方法。原理上跟递归看起来很像，但它的特点是不重复计算子问题，而是利用之前子问题的结果直接计算当前子问题，其结果又存储起来供下一子问题利用，用空间换取时间。动态规划是一种自底向上的方法，而递归是一种自顶向下的方法。

适用于动态规划的问题有以下几个特点：

  1. 当前子问题的解依赖于之前子问题的解；
  2. 只要求最值、判定是否或求方案总数，而不要求所有过程中的结果；
  3. 输入量通常是序列或者字符串，而不是集合；

如何构建动态规划？

关键在于定义问题的状态和找出状态转移方程。“状态”就是子问题的解，“状态转移方程”就是递推关系式，描述如何从当前解得到下一步的解。写动态规划代码的步骤为：

  1. 定义状态；
  2. 设定初始值；
  3. 在循环中用状态转移方程不断迭代得到子问题的解直至解决总问题。

常见的动态规划类型

坐标型

输入量通常是二维坐标。典型题目为Unique Path和Triangle。此类型不需要在定义状态时加一行一列，也因此需要考虑输入为None的处理，优化空间时，采用rolling list。
例题：

有一个机器人的位于一个 m × n 个网格左上角。 机器人每一时刻只能向下或者向右移动一步。机器人试图达到网格的右下角。网格中的障碍和空位置分别用1和0来表示。问有多少条不同的路径？

单序列型

典型问题为Maximum Product Subarray，输入量通常是一个序列，定义状态时常在开头增加一项，也因而不需要额外处理输入为None的情形。空间优化通常用一个变量代替一个列表，可从$O(n)$优化至$O(1)$。

双序列型

典型问题为Edit Distance和Word Break，通常是研究两个序列或字符串之间的关系，基础情形下将状态定义成二维的，常增加一行一列来设置初始值和处理None输入。空间优化通常是用rolling list或两个lists将额外空间复杂度从$O（n^2)$降至$O(n)$。
例题：

Given two words word1 and word2, find the minimum number of steps required to convert word1 to word2. (each operation is counted as 1 step.)
You have the following 3 operations permitted on a word:
Insert a character
Delete a character
Replace a character
Example
Given word1 = “mart” and word2 = “karma”, return 3.

背包型

典型问题为0/1背包问题，

有N件物品和一个容量为V的背包。第i件物品的费用是c[i]，价值是w[i]。求解将哪些物品装入背包可使价值总和最大。

处理步骤与双序列相似。背包问题属于NP-Compelete问题,参考背包问题九讲，背包问题的状态转移方程为 $f[i][v]=max{f[i-1][v],f[i-1][v-c[i]]+w[i]}$。在采用rolling list优化空间复杂度时，需将内循环顺序反过来，伪代码如下：

for i=1..N
for v=V..0
    f[v]=max{f[v],f[v-c[i]]+w[i]};
