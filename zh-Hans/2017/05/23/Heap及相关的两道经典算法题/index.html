Heap及相关的两道经典算法题

发表于 2017-05-23 | 分类于 Academy

什么是Heap?

Heap是树的一个分支，其特点是父节点的key总是比子节点的key大（或者小）。Heap也有很多子类，最常用的是binary heap, 它除了heap的特点，最重要的是形态上属于complete binary tree。也就是说，除了最底层，每一层的节点数都是满的，而最底层的节点都在尽可能的左侧。

Heap与priority queue

Priority queue是从queue的概念引申过来的，queue是指先进先出的一种数据结构，priority queue则是按照数据的优先级来存储数据，而不是数据的先后顺序。需要注意的是，priority queue是一种抽象概念的数据结构，描述的是按照优先级存储数据的功能。binary heap作为一种具体的数据结构，是实现priority queue的最广泛最高效的选择。

Binary Heap的构建

Binary heap具有complete binary tree的结构，因此可以用array来实现，这是因为对于任一个节点pos，其子节点的坐标为$2pos+1$和$2pos+2$。具体的实现可以参照这一篇博文Python中heapq源码分析。

两个与heap相关的经典算法题

一、找到数组中第K大的数

给定一个数组nums和一个整数k， 找到nums中第K大的数。比如nums = [11, 12, 3, 14, 15, 6, 5, 2, 3], k = 3， 则返回12。
几种思路：

  1. 选择一种排序算法将nums完全排序，则时间复杂度最好可为O（N*log(N))；
  2. 用冒泡排序，但是只循环k次，返回nums[len(nums)-k];
  3. 用min heap维护一个大小固定为k的heap，再将nums中k以后的元素循环移入，则最后得到的heap中最末尾元素为所求；
  4. 借用quick sort思想，因为只需要找到第k大的数，所以只要确保k之前的元素都不小于它，而k之后的元素都不大于它，如此，当pivot = k时终止quick sort即可。

另外，用max heap也是一种思路。但是需要注意亮点，一是python中没有提供max heap，二是需要将nums整体维护heap结构（$O(n)$）,再弹出k次得到结果（$O(klog(n))$）。因此总的时间复杂度为$O(n)+O(klog(n))$

最后是思路4，quick sort的思想。具体来说，调用partition函数取数组中末尾元素作为pivot value，通过比较将其交换放置在一个位置，使得pivot前的元素都不小于它，pivot后的元素都不大于它。通过quickSort函数不断调用partition函数，直到找到一个pivot value的坐标等于k。
时间复杂度上，每一次调用quickSort后，循环减半，因此 $ average\ time = n + n/2 + n/4 + … = O(n)$。但是在最坏情况下（比如所有元素都相同，k又极大），时间复杂度可达到 $O(n^2)$。空间复杂度上，因为是in-place排序，没有额外使用空间，因此为 $O(1)$

总结起来，这道题用min heap和quick sort来解是比较理想的，quick sort的平均时间复杂度更好，但是代码更复杂。min heap的代码简洁，时间复杂度也不错。
